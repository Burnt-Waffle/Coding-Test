# Coding-Test

### 파이썬을 사용하는 이유

코딩테스트에서 파이썬이 다른 언어(C++, Java 등)에 비해 가장 큰 유리한 점은 간결한 문법을 바탕으로 한 압도적인 구현 생산성과 풍부한 내장 라이브러리에 있다. 이 덕분에 제한된 시간 내에 알고리즘을 설계하고 코드로 옮겨야 하는 코딩테스트 환경에서 파이썬이 가장 유리하다. 주요 장점은 다음과 같습니다.

1. 간결하고 직관적인 문법
   - 짧은 코드: C++로 20~30줄 작성해야 할 코드를 파이썬은 5~10줄 내외로 구현할 수 있다.
   - 가독성: 영어 문장과 유사하여 코드를 읽기 쉽고, 들여쓰기가 필수라 코드 구조가 명확하다.
   - 세미콜론(`;`)과 중괄호(`{}`) 불필요: 코드 작성 시 괄호 실수나 문장 끝 처리에 신경 쓸 필요가 없어 구현 속도가 빠르다.

2. 강력한 내장 라이브러리 및 자료구조
   - `collections` 모듈: `deque`(양방향 큐), `Counter`(요소 개수 세기) 등 알고리즘 문제에서 자주 쓰이는 자료구조를 즉시 활용 가능하다.
   - `heapq`, `bisect`, `itertools`: 우선순위 큐(힙), 이진 탐색, 순열/조합 등을 구현 없이 라이브러리 함수 호출만으로 사용할 수 있다.
   - `sort()`, `sorted()`: 정렬 함수가 매우 강력하며, 람다 식을 이용한 다중 조건 정렬이 매우 쉽다.

3. 유연한 자료형 처리동적 타이핑: 변수 타입을 미리 선언할 필요가 없어 코딩이 빠르다.
   - 무한대 정수 지원: 정수형(`int`)의 크기 제한이 사실상 없어, 다른 언어에서 오버플로우를 걱정해야 하는 큰 수 계산 문제를 고민 없이 풀 수 있다.
   - 리스트와 딕셔너리: 파이썬의 리스트는 다른 언어의 배열과 벡터가 혼합된 형태이며, 딕셔너리는 해시맵을 매우 간결하게 대체한다.

4. 문자열 및 리스트 슬라이싱
   - 파이썬의 `arr[a:b:c]`와 같은 슬라이싱 기능은 문자열 뒤집기, 특정 부분 추출 등을 한 줄로 해결하게 해줍니다. (예: `s[::-1]`로 문자열 뒤집기)
